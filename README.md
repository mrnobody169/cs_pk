Để chốt nghiệm thu, anh cần em xác nhận rõ ràng, từng ý một, các nội dung sau:

1. Về phạm vi công việc:
Em xác nhận toàn bộ các hạng mục thuộc Milestone 1 và Milestone 2 đã được triển khai xong đầy đủ đúng theo hợp đồng, không còn hạng mục nào đang làm dở, làm thiếu, hoặc để lại cho milestone sau.

2. Về Milestone 1 – tài liệu & thiết kế:
Em xác nhận đã bàn giao cho anh toàn bộ hồ sơ và tài liệu dự án bên em đang dùng để triển khai, bao gồm:
- PRD chi tiết cho từng game
- Tài liệu luật chơi, odds và luồng xử lý thắng thua
- Thiết kế database / database schema / DB design
- Danh sách API hệ thống và API cho thuê game (Swagger/Postman)
- File thiết kế giao diện (Figma hoặc HTML)

Các tài liệu này là bản chính thức đang dùng để phát triển hệ thống. Ngoài những tài liệu trên, bên em không còn giữ lại tài liệu nào khác liên quan đến dự án mà chưa bàn giao cho anh.

3. Em xác nhận toàn bộ source code của Milestone 1 và Milestone 2 đã được đưa đầy đủ lên repo Git do bên anh quản lý.
3.1. Hệ thống User (core)
•	User có thể đăng ký, đăng nhập
•	Có phân quyền ở mức core (user / admin / agent hoặc tương đương)
•	Backend đã có cơ chế kiểm soát quyền truy cập theo role
(chưa yêu cầu UI admin đầy đủ ở giai đoạn này)
3.2. Wallet Engine
•	Có chức năng cộng/trừ tiền
•	Có lịch sử giao dịch
•	Xử lý giao dịch theo nguyên tắc an toàn, không trừ tiền sai
3.3. Frontend
•	Frontend sảnh game hoạt động
•	Frontend trang game hoạt động để test nghiệp vụ
3.4. Đặt cược & khóa cược
•	User có thể đặt cược
•	Hệ thống khóa cược đúng thời điểm theo thời gian server
•	Không cho phép đặt cược sau khi đã khóa cược
3.5. Game Gateway
•	Đã triển khai Game Gateway ở mức nền tảng
•	Đủ để làm cơ sở tích hợp game và đối tác ở các milestone tiếp theo
3.6. Môi trường STAGING
•	STAGING đã được deploy đầy đủ frontend, backend và database
•	Bên A có thể tự truy cập, tự test, không phụ thuộc vào môi trường nội bộ của Bên B

4. Em xác nhận hệ thống User ở Milestone 2 đã bao gồm:
- User đăng ký, đăng nhập
- Nền tảng phân quyền ở backend cho user / admin / agent
Đủ để triển khai các chức năng admin ở Milestone 3 mà không phải làm lại backend.

5. Em xác nhận các chức năng wallet, frontend sảnh game, frontend trang game, đặt cược – khóa cược và game gateway đã hoạt động trên môi trường STAGING để phục vụ nghiệm thu.

6. Về frontend, anh kiểm tra thực tế trên mobile và thấy giao diện hiện tại chưa responsive, hiển thị và thao tác chưa đúng. Nội dung này hiện chưa đạt yêu cầu nghiệm thu. Em rà soát, khắc phục và xác nhận lại rõ cho anh.

7. Em xác nhận môi trường STAGING cho phép anh tự truy cập, tự test và tự tiếp quản hệ thống mà không phụ thuộc vào môi trường nội bộ của bên em.


Căn cứ các nội dung trên, việc nghiệm thu Milestone 1 và Milestone 2 được thực hiện trên cơ sở:
•	Công việc đã hoàn thành đúng phạm vi hợp đồng
•	Tài liệu và source code đã được bàn giao đầy đủ

Anh cần em xác nhận rõ ràng các nội dung trên. Chỉ sau khi em xác nhận đầy đủ từng ý và khắc phục xong các nội dung chưa đạt, anh mới tiến hành nghiệm thu và xử lý thanh toán theo hợp đồng.
